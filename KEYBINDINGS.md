### 方法 1: 在 WezTerm 中查看（推荐）
按 `F2` 打开命令面板，搜索 `show-keys` 即可查看所有快捷键

### 方法 2: 命令行查看
```bash
wezterm show-keys --lua
```

```bash
cat ~/.config/wezterm/KEYBINDINGS.md
# 或在编辑器中打开
```

## 📌 完整快捷键详解

### 🎬 基础操作 - F 键系列

#### `F1` - 进入复制模式
- **作用**：进入复制模式，可以用 Vim 快捷键选择和复制文本
- **使用场景**：需要精确选择并复制一段文本
- **退出方式**：按 `Ctrl+C` 或 `Escape`
- **详见**：[复制模式快捷键](#-复制模式快捷键-f1-进入)

#### `F2` - 打开命令面板
- **作用**：显示所有可用命令和操作的搜索面板
- **使用场景**：找不到快捷键时搜索功能；也可以执行各种命令
- **搜索功能**：支持中英文搜索（如搜索"分割"可找到 SplitPane）
- **提示**：这是最强大的帮助工具，记住它可以减少记忆负担

#### `F3` - 显示启动器
- **作用**：显示所有可用的启动选项（新建标签、域、工作区等）
- **使用场景**：需要快速启动新标签或切换域
- **注意**：会显示所有配置的域（如 WSL 域、SSH 主机等）

#### `F4` - 启动器（仅标签页）
- **作用**：快速新建标签页
- **使用场景**：只需要新建标签，不需要看其他选项
- **加速**：比 `F3` 响应更快

#### `F5` - 启动器（仅工作区）
- **作用**：显示并快速切换或新建工作区
- **使用场景**：管理多个独立的工作区
- **工作区概念**：可以将多个窗口/标签组织为一个工作区

#### `F11` - 切换全屏
- **作用**：让 WezTerm 窗口进入或退出全屏模式
- **使用场景**：需要最大化显示空间时
- **注意**：全屏时窗口会隐藏标题栏和任务栏

#### `F12` - 显示调试信息
- **作用**：打开 WezTerm 的内部调试覆盖层
- **使用场景**：诊断问题、查看性能指标、调试配置
- **包含信息**：GPU 渲染情况、帧率、内存使用等

### ✏️ 文本操作 - 复制/粘贴/搜索

#### `Ctrl+Shift+C` - 复制
- **作用**：复制选中的文本到剪贴板
- **使用场景**：快速复制终端中的内容
- **与标准快捷键的关系**：标准 `Ctrl+C` 用于中断进程，此键避免冲突
- **提示**：WezTerm 支持自动复制选中内容，可在配置中启用

#### `Ctrl+Shift+V` - 粘贴
- **作用**：粘贴剪贴板中的内容到当前光标位置
- **使用场景**：粘贴命令、路径或其他文本
- **与标准快捷键的关系**：与复制快捷键配套，避免与 `Ctrl+V` 冲突

#### `Alt+F` - 搜索文本
- **作用**：打开搜索框，可以在屏幕历史中搜索文本
- **使用场景**：查找之前出现过的错误信息、日志输出等
- **搜索范围**：搜索整个滚动缓冲区
- **搜索方式**：支持正则表达式搜索
- **导航**：找到后可以用上下方向键导航到其他匹配项

#### `Alt+Ctrl+U` - 快速选择 URL
- **作用**：识别并快速选择屏幕上的 URL，可以直接打开
- **支持的格式**：
  - 括号中的 URL: `(https://example.com)`
  - 方括号中的 URL: `[https://example.com]`
  - 花括号中的 URL: `{https://example.com}`
  - 尖括号中的 URL: `<https://example.com>`
  - 纯 URL: `https://example.com`
- **操作**：按后会显示输入框，选择要打开的 URL
- **使用场景**：快速打开日志、错误信息中的链接

### 🔀 光标移动 - Alt+方向键/Backspace

#### `Alt+←` - 光标移动到行首
- **作用**：将光标快速移动到当前行的开头
- **VT100 代码**：发送 ESC+O+H 序列
- **使用场景**：需要快速回到行首编辑
- **vs 标准快捷键**：比 `Home` 键更快的替代方案

#### `Alt+→` - 光标移动到行末
- **作用**：将光标快速移动到当前行的结尾
- **VT100 代码**：发送 ESC+O+F 序列
- **使用场景**：需要快速到行末追加内容
- **vs 标准快捷键**：比 `End` 键更快的替代方案

#### `Alt+Backspace` - 删除整行
- **作用**：删除当前行的所有内容
- **发送信号**：Ctrl+U（标准 Unix 行删除命令）
- **使用场景**：输入错误，需要快速清空当前行
- **提示**：这会发送给 Shell，而非 WezTerm 自身处理

### 📑 标签页操作 (Tab Management)

#### `Alt+T` - 新建标签页
- **作用**：创建一个新的标签页（使用默认域）
- **默认域**：DefaultDomain（通常是本地 Shell）
- **使用场景**：需要额外的工作空间
- **位置**：新标签在当前标签的右侧
- **键盘焦点**：焦点自动移动到新标签

#### `Alt+Ctrl+T` - 新建 WSL 标签页
- **作用**：创建一个新的 WSL Ubuntu 标签页（带 Fish Shell）
- **目标域**：`wsl:ubuntu-fish`（Linux 工作环境）
- **使用场景**：需要 Linux 环境时快速开启
- **vs 标准新建**：比 `Alt+T` 后再选择域更快
- **前置条件**：需要安装 WSL 和 Ubuntu

#### `Alt+[` - 切换到上一个标签页
- **作用**：激活当前标签的左侧标签
- **循环**：到了最左边的标签后会循环到最右边
- **使用场景**：快速在标签间切换
- **快速导航**：可多次按压快速浏览

#### `Alt+]` - 切换到下一个标签页
- **作用**：激活当前标签的右侧标签
- **循环**：到了最右边的标签后会循环到最左边
- **使用场景**：快速在标签间切换
- **快速导航**：可多次按压快速浏览

#### `Alt+Ctrl+[` - 向左移动标签页
- **作用**：将当前标签页向左移动（交换位置）
- **使用场景**：重新组织标签的顺序
- **限制**：不能超过最左边

#### `Alt+Ctrl+]` - 向右移动标签页
- **作用**：将当前标签页向右移动（交换位置）
- **使用场景**：重新组织标签的顺序
- **限制**：不能超过最右边

#### `Alt+0` - 重命名标签页
- **作用**：为当前标签页设置自定义名称
- **输入方式**：弹出输入框，输入新名称后按 Enter
- **使用场景**：给不同工作区标注有意义的名称（如"后端"、"前端"）
- **覆盖**：会覆盖自动生成的标签名称

#### `Alt+Ctrl+0` - 撤销重命名
- **作用**：恢复标签页的自动生成名称
- **自动名称**：基于当前运行的命令
- **使用场景**：之前手动重命名，现在想回到自动名称

#### `Alt+9` - 显示/隐藏标签栏
- **作用**：切换标签栏的可见性
- **全屏效果**：隐藏标签栏可获得更多垂直空间
- **使用场景**：只有一个标签时可隐藏以节省空间
- **恢复**：再按一次 `Alt+9` 可恢复显示

#### `Alt+Ctrl+W` - 关闭标签页
- **作用**：关闭当前标签页
- **确认**：默认不需要确认（如果标签是干净的）
- **警告**：如果标签有未完成的进程可能会提示确认
- **焦点**：关闭后焦点会移动到相邻的标签页

### 📦 窗格操作 (Pane Management)

#### `Alt+\` - 竖向分割窗格
- **作用**：将当前窗格竖向分成两个（新窗格在右侧）
- **效果**：原窗格宽度不变，高度不变，左边是原窗格，右边是新窗格
- **域**：新窗格使用当前窗格相同的域
- **使用场景**：在两个命令行之间对比工作
- **多次分割**：可以多次使用实现更复杂的布局

#### `Alt+Ctrl+\` - 横向分割窗格
- **作用**：将当前窗格横向分成两个（新窗格在下方）
- **效果**：原窗格宽度不变，高度不变，上面是原窗格，下面是新窗格
- **域**：新窗格使用当前窗格相同的域
- **使用场景**：同时查看多个进程输出，如编译和日志
- **多次分割**：可以多次使用实现网格布局

#### `Alt+Enter` - 缩放/取消缩放窗格
- **作用**：让当前窗格临时全屏显示，再按一次恢复
- **使用场景**：需要专注于某个窗格，或查看完整输出
- **快速切换**：可快速在全屏和分割视图间切换
- **其他窗格**：缩放时其他窗格被隐藏但仍在运行

#### `Alt+W` - 关闭当前窗格
- **作用**：关闭当前窗格（如果还有其他窗格）
- **单窗格限制**：如果只有一个窗格则无法关闭
- **焦点移动**：关闭后焦点移动到相邻窗格
- **确认**：通常不需要确认（除非有运行中的进程）

#### `Alt+Ctrl+K` - 上方活跃窗格
- **作用**：激活（聚焦）上方的窗格
- **如果不存在**：不执行任何操作
- **使用场景**：快速在多个窗格间切换焦点
- **Vim 导航**：K 代表上方（Vim 习惯）

#### `Alt+Ctrl+J` - 下方活跃窗格
- **作用**：激活（聚焦）下方的窗格
- **如果不存在**：不执行任何操作
- **使用场景**：快速在多个窗格间切换焦点
- **Vim 导航**：J 代表下方（Vim 习惯）

#### `Alt+Ctrl+H` - 左方活跃窗格
- **作用**：激活（聚焦）左方的窗格
- **如果不存在**：不执行任何操作
- **使用场景**：快速在多个窗格间切换焦点
- **Vim 导航**：H 代表左方（Vim 习惯）

#### `Alt+Ctrl+L` - 右方活跃窗格
- **作用**：激活（聚焦）右方的窗格
- **如果不存在**：不执行任何操作
- **使用场景**：快速在多个窗格间切换焦点
- **Vim 导航**：L 代表右方（Vim 习惯）

#### `Alt+Ctrl+P` - 窗格选择器
- **作用**：显示所有窗格的编号选择器，可快速切换并交换
- **选择方式**：点击数字（0-9）或按对应数字键
- **交换模式**：使用 `SwapWithActiveKeepFocus` 确保焦点留在当前位置
- **使用场景**：有很多窗格时快速定位到某个窗格
- **可视化**：每个窗格都显示一个大数字方便选择

#### `Alt+U` - 向上滚动 5 行
- **作用**：在当前窗格中向上滚动 5 行
- **不进入复制模式**：只是滚动查看历史输出
- **使用场景**：快速查看刚才输出的内容
- **连续使用**：可多次按压查看更早的输出

#### `Alt+D` - 向下滚动 5 行
- **作用**：在当前窗格中向下滚动 5 行
- **回到最新**：如果滚动到最底部会看到最新输出
- **使用场景**：滚回到最新的输出
- **连续使用**：可多次按压快速滚回

#### `PageUp` - 向上翻页
- **作用**：滚动一整页向上（约 18-20 行取决于窗口高度）
- **快速滚动**：比 `Alt+U` 更快
- **使用场景**：快速查看很久之前的输出
- **标准按键**：这是标准的终端翻页键

#### `PageDown` - 向下翻页
- **作用**：滚动一整页向下
- **回到底部**：连续按可快速回到最新输出
- **使用场景**：从历史输出快速回到当前
- **标准按键**：这是标准的终端翻页键

### 🪟 窗口操作 (Window Management)

#### `Alt+N` - 新建窗口
- **作用**：创建一个全新的 WezTerm 窗口
- **独立性**：每个窗口有自己的标签和布局
- **使用场景**：在多个显示器上工作，或保持不同项目分开
- **独立运行**：各窗口独立，互不影响

#### `Alt+=` - 增大窗口尺寸
- **作用**：将 WezTerm 窗口宽高各增加 50 像素
- **增量**：每次增加 50px × 50px
- **限制**：全屏时此快捷键无效
- **使用场景**：逐步调整窗口大小
- **快速调整**：可连续按压快速增大

#### `Alt+-` (Alt+减号) - 减小窗口尺寸
- **作用**：将 WezTerm 窗口宽高各减少 50 像素
- **减量**：每次减少 50px × 50px
- **限制**：全屏时此快捷键无效
- **最小值**：不能减小到看不见的尺寸
- **快速调整**：可连续按压快速缩小

#### `Alt+Ctrl+Enter` - 最大化窗口
- **作用**：最大化 WezTerm 窗口（与全屏不同）
- **差异**：最大化保留标题栏和任务栏，全屏隐藏
- **恢复**：再按一次可恢复原大小
- **使用场景**：需要大视图但保留操作系统界面
- **vs 全屏**：比 F11 全屏更灵活

### 🖼️ 背景图片控制

#### `Alt+/` - 随机更换背景
- **作用**：从所有配置的背景中随机选择一个
- **速度**：即时切换
- **使用场景**：随意更换背景增加新鲜感
- **图片来源**：来自 `~/.config/wezterm/backdrops/` 目录
- **连续使用**：可多次按压直到找到喜欢的背景

#### `Alt+,` - 切换到前一个背景
- **作用**：循环向前导航背景列表
- **顺序**：按背景文件列表顺序
- **循环**：到了最后一个会跳回第一个
- **使用场景**：系统地浏览所有背景
- **记住喜欢的**：可以在配置中设置默认背景

#### `Alt+.` - 切换到后一个背景
- **作用**：循环向后导航背景列表
- **顺序**：按背景文件列表顺序
- **循环**：到了第一个会跳到最后一个
- **使用场景**：系统地浏览所有背景
- **配合前一个**：可以快速找到想要的背景

#### `Alt+Ctrl+/` - 背景选择器
- **作用**：显示所有可用背景的列表，可搜索和选择
- **搜索功能**：支持模糊搜索背景名称
- **预览**：显示背景名称但不显示缩略图
- **使用场景**：有很多背景时快速查找特定的
- **选择方式**：按上下方向键或输入来搜索

#### `Alt+B` - 切换焦点模式
- **作用**：为背景启用/禁用焦点模式
- **焦点模式**：变暗非活跃区域，突出活跃区域
- **目的**：提高视觉焦点，减少视觉干扰
- **使用场景**：有多个窗格时，突出显示当前活跃窗格
- **视觉效果**：背景或透明度会根据焦点改变

### 🔗 LEADER 键扩展命令

**LEADER 键定义**：`Alt+Ctrl+Space`

#### LEADER 键的工作原理
- **激活方式**：按 `Alt+Ctrl+Space`
- **等待输入**：激活后 WezTerm 会进入命令等待模式
- **超时**：3 秒内没有按键会自动退出
- **二级命令**：按下 F 或 P 进入不同的操作模式
- **视觉反馈**：状态栏通常会显示 "LEADER" 表示激活

---

#### `LEADER+F` - 字体大小调整

**激活序列**：
```
1. 按 Alt+Ctrl+Space (进入 LEADER 模式)
2. 按 F (进入字体调整模式)
3. 按下述键进行操作
```

**可用命令**（进入后）：

| 键 | 作用 | 可重复 |
|----|------|--------|
| `K` | 增大字体 | ✅ 是 |
| `J` | 减小字体 | ✅ 是 |
| `R` | 重置字体到默认大小 | ✅ 是 |
| `Escape` | 退出字体调整模式 | ❌ |
| `Q` | 退出字体调整模式 | ❌ |

**使用示例 1**：增大字体 2 档
```
1. 按 Alt+Ctrl+Space
2. 看到 LEADER 激活后，按 f
3. 快速按两次 k (K K)
4. 按 Escape 或 q 退出
```

**使用示例 2**：重置字体
```
1. 按 Alt+Ctrl+Space + f
2. 按 r
3. 按 q
```

**使用示例 3**：快速增大多档
```
1. 按 Alt+Ctrl+Space
2. 按 f
3. 连续快速按 k k k k k (可以不松开)
4. 按 Escape
```

**特点**：
- 可持续按压同一键快速调整
- 字体大小改变即时反映
- 全窗口应用，所有窗格字体同时改变
- 3 秒超时，时间充足不用急

---

#### `LEADER+P` - 窗格大小调整

**激活序列**：
```
1. 按 Alt+Ctrl+Space (进入 LEADER 模式)
2. 按 P (进入窗格调整模式)
3. 按下述键调整尺寸
```

**可用命令**（进入后）：

| 键 | 作用 | 效果 | 可重复 |
|----|------|------|--------|
| `K` | 向上扩展 | 当前窗格向上增高，下方窗格缩小 | ✅ 是 |
| `J` | 向下扩展 | 当前窗格向下增高，上方窗格缩小 | ✅ 是 |
| `H` | 向左扩展 | 当前窗格向左增宽，右方窗格缩小 | ✅ 是 |
| `L` | 向右扩展 | 当前窗格向右增宽，左方窗格缩小 | ✅ 是 |
| `Escape` | 退出调整模式 | 保存当前大小 | ❌ |
| `Q` | 退出调整模式 | 保存当前大小 | ❌ |

**使用示例 1**：让右边的窗格更宽
```
1. 按 Alt+Ctrl+Space
2. 按 p
3. 按 3 次 l (l l l)
4. 按 Escape
```

**使用示例 2**：给上方窗格增加高度
```
1. Alt+Ctrl+Space → p
2. 按 5 次 k
3. q
```

**使用示例 3**：平衡三个窗格大小
```
1. Alt+Ctrl+Space → p
2. 按 k k (调整高度)
3. 按 l l (调整宽度)
4. Escape
```

**使用场景**：
- 编写代码（左宽）+ 日志输出（右窄）
- 多个终端混合大小布局
- 临时调整尺寸以适应内容

**特点**：
- 可连续按压快速调整
- 每按一次调整 1 像素单位
- 仅影响当前激活窗格（相邻窗格相应缩小）
- 调整实时预览
- 不能调整到小于最小尺寸

## ⚙️ 自定义快捷键

所有快捷键定义在：`~/.config/wezterm/config/bindings.lua`

### 修改步骤

1. **编辑配置文件**
   ```bash
   vim ~/.config/wezterm/config/bindings.lua
   ```

2. **重新加载配置**
   - 方式 1：重启 WezTerm
   - 方式 2：按 `F2` 打开命令面板，搜索 "reload config"

3. **验证更改**
   - 按 `F2` → 搜索 "show-keys" 验证新快捷键

4. **提交到 GitHub**（如果使用版本控制）
   ```bash
   cd ~/.config/wezterm
   git add config/bindings.lua
   git commit -m "Update keybindings"
   git push
   ```

### 快捷键语法

```lua
{
    key = 'a',              -- 按键
    mods = 'CTRL|SHIFT',    -- 修饰键
    action = act.SendString('hello')  -- 执行的动作
}
```

## 💡 常见问题 (FAQ)

**Q: 为什么我的快捷键不工作？**
A: 
1. 检查是否在正确的模式（如复制模式、LEADER 模式）
2. 按 `F2` → "show-keys" 验证快捷键是否存在
3. 确认修饰键正确（Windows/Linux 是 Alt，不是 Super）

**Q: 可以修改快捷键吗？**
A: 可以！编辑 `~/.config/wezterm/config/bindings.lua`，然后重启 WezTerm 或用命令面板重新加载配置

**Q: LEADER 键超时了怎么办？**
A: 再按一次 `Alt+Ctrl+Space` 重新激活，3 秒内有足够的时间

**Q: 复制模式中如何快速找到内容？**
A: 按 `/` 搜索，输入关键词，按 `n` 找下一个匹配

**Q: 为什么有些快捷键不响应？**
A: 可能被系统或其他应用占用，改用命令面板（F2）或修改快捷键
